Curalink — AI Medical Research Assistant Hackathon
________________


🎯 Overview
Build a full-stack AI-powered Medical Research Assistant Prorotype using the MERN stack, powered by a custom open-source LLM.
This system should act as a health research companion that:
Understands user context → retrieves high-quality medical research → reasons over it → delivers structured, personalized, and source-backed answers.
This is not just a chatbot — it is a research + reasoning system.
________________


💡 Objective
Design a system that can:
* Accept user medical context (disease, intent, optional location)
* Fetch relevant research publications and clinical trials
* Deliver structured, high-quality, research-backed answers
* Maintain context across conversations
* Provide deep, personalized insights instead of generic responses
________________


🧠 Core Expectations
1️⃣ Structured Input Understanding + Query Expansion
Your system must handle both structured and natural queries.
Example Input
* Patient Name: John Smith
* Disease of Interest: Parkinson’s disease
* Additional Query: Deep Brain Stimulation
* Location: Toronto, Canada
________________


Expected Behavior
The system should automatically expand the query intelligently.
👉 Example:
Instead of searching:
“deep brain stimulation”
It should search:
“deep brain stimulation + Parkinson’s disease”
________________


Output Should Combine
* Relevant publications
* Relevant clinical trials
👉 All results must be context-aware and merged intelligently

Publications (OpenAlex + PubMed)

They represent:
* Real-world validation of treatments

📄 API Query Endpoints (ClinicalTrials + PubMed + OpenAlex)

________________

🧪 ClinicalTrials.gov API v2

 Use structured parameters for more accurate results.

🔹 Base Endpoint

 <https://clinicaltrials.gov/api/v2/studies>
 🔹 Basic Query Template

 <https://clinicaltrials.gov/api/v2/studies?query.cond=><DISEASE>&filter.overallStatus=<STATUS>&pageSize=<SIZE>&format=json
 🔹 Example (Diabetes)

 <https://clinicaltrials.gov/api/v2/studies?query.cond=diabetes&filter.overallStatus=RECRUITING&pageSize=20&format=json>
________________


📚 PubMed (NCBI)

 🔹 Step 1: Search (Get IDs)

 <https://eutils.ncbi.nlm.nih.gov/entrez/eutils/esearch.fcgi?db=pubmed&term=><QUERY>&retmax=<SIZE>&sort=pub+date&retmode=json
 🔹 Step 2: Fetch Details

 <https://eutils.ncbi.nlm.nih.gov/entrez/eutils/efetch.fcgi?db=pubmed&id=><ID1,ID2,...>&retmode=xml
________________


🔹 Example (Lung Cancer)

 Step 1: Search

 <https://eutils.ncbi.nlm.nih.gov/entrez/eutils/esearch.fcgi?db=pubmed&term=lung+cancer+AND+clinical+trial&retmax=3&sort=pub+date&retmode=json>
 Example Response (Simplified)

 {
  "esearchresult": {
    "idlist": ["41732954", "41949826", "41915379"]
  }
}
 Step 2: Fetch

 <https://eutils.ncbi.nlm.nih.gov/entrez/eutils/efetch.fcgi?db=pubmed&id=41732954,41949826,41915379&retmode=xml>
 👉 Returns full article data (title, abstract, authors, journal, etc.)

________________

🌐 OpenAlex

 🔹 Base Endpoint

 <https://api.openalex.org/works>
 🔹 Basic Template

 <https://api.openalex.org/works?search=><QUERY>&per-page=<SIZE>&page=<PAGE>&sort=<SORT>
________________


🔹 Example (Lung Cancer Research)

 Relevance (default):

 <https://api.openalex.org/works?search=lung+cancer+research&per-page=25&page=1&sort=relevance_score:desc>
 Newest First:

 <https://api.openalex.org/works?search=lung+cancer+research&per-page=25&page=1&sort=publication_date:desc>
 With Date Filter:

 <https://api.openalex.org/works?search=lung+cancer+research&per-page=25&page=1&sort=relevance_score:desc&filter=from_publication_date:2020-01-01,to_publication_date:2025-12-31>
________________
